How to Work with Git and GitHub

2024-03-21

Normi Chen

Git

Git is like the superhero of coding tools. It helps developers keep track of changes, collaborate with teammates, and work on projects seamlessly. With its decentralized nature, Git allows offline work and smooth synchronization with team efforts. It's the backbone of modern software development, keeping code organized and projects running smoothly. As follows, we will show that, how to use git to track the code.

Initialization

Configure the project folder to allow Git to do version tracking. Generally, any folders are not work for git, so we should do this at the start. The steps as follows:

  1. Create a project folder.
  2. Use the terminal to switch the path to the project path cd path/folderName.
  3. Input git init.

Tracking status

For this step we can observe which files in this directory have not been tracked or have not been updated.

  • Input git status.

Add files

When we create some new files or edits some old files, we hope that it can be tracked. As follows is two ways to add files to the tracking list. This step indicating that you want Git to record versions of these files.

  • Input git add fileName.
  • Input git add . (dot means adding all files from this folder).

Commitment

After making any changes, deletions, or additions to this project directory, you should commit the version once. Then git will record the version of files from tracking list. If you want, you can type some description for this commit. That will easer to understaning that what' s the chang of this version.

  • Input git commit -m "description for this commitment".
When commit for the first time, you will be asked to enter your name and email in order to record who created this commitment.

View version details

Displays the version information recorded by this project, including: author, date, time, and remarks.

  • Input git log.

Branch

Git allows the creation of different branches, allowing users to copy the original data, and modifications on the branch will not affect the original branch. It is usually used to develop new features.

  • Input git branch.

Process after coding

After coding, you can type the code as blow in your terminal. Then git will record the version for this update. And you can also check the status for this folder.

git status git add . git commit -m "description" git status git log

GitHub

GitHub is basically like a social network for coders. It's a place where developers can store their code, collaborate with others, and showcase their projects to the world. Think of it as a virtual workspace where teams can work together on coding projects, track changes, and manage versions of their code. Plus, it's not just for work - many developers use GitHub to contribute to open-source projects and build their coding portfolios. It's the go-to platform for modern software development, making teamwork and sharing code a breeze.

Repository of remote

This step is used to confirm whether this folder is linked to the repository on GitHub.

  • Input git remote -v (v represent to display the information).

Upload

As follows, we will display how to push the project into GitHub repository. Then the project will backup in GitHub.

Connection

To connect the local project to the remote project on GitHub, you only need to make this connection for the first time, which is similar to the initialization of Git.

  1. Create a repository in your GitHub (if you don' t have the remote project for this local project yet).
  2. Copy the url for the repository.
  3. Input git remote add remoteName url.
The remoteName is usually written as origin. You can choose it by yourself. But the system default is origin, so everyone always write origin for the remoteName.

Push

Update the locally modified version to the project space of the linked GitHub, which is equivalent to the downloaded pull.

  • Input git push remoteName branchName.
If you are using it for the first time, the system will ask for your GitHub username and password in order to verify your identity. If your repository dose not have any branch then the branchName will be main.

General workflow

For the new project, after coding you can type the below code to connect the local project to the remote project.

git status git add . git status git commit -m "description" git status git remote -v git remote add origin url git remote -v git push origin main

For the old project, after coding you can type the below code to update the new version of this project.

git status git add . git status git commit -m "description" git status git push origin main

Download

In this section, we will show that how to clone or pull the remote project into local path. Generally, we should do this before start coding, for confirm that our code is newest version.

Clone

When there is no project on the local side, download the remote project to your own machine for the convenience of editing programs and collaborating with others.

  1. Use the terminal to switch the path to the folder that you want to save the project.
  2. Input git clone url folderName.
There is no need to have a project folder on the local side. The folderName is mean the name of your local project folder. After clone the repository, the local project is already connected to the remote project, so you can see the information through git remote -v.

Pull

Before every time you start editing a project, we should merge and update the latest remote project to your own local project, and then continue editing the program, which is equivalent to uploading push.

  • Input git pull remoteName branchName.

General workflow

For the new project, before coding you can type the below code to clone the remote project to the local project.

git clone url folderName git remote -v git status git add . git status git commit -m "description" git status git remote -v git push origin main

For the old project, before coding you can type the below code to update the new version of this project in your local project folder.

git pull origin main git status git status git add . git status git commit -m "description" git status git push origin main

The above is the most basic operation method of Git and GitHub. We can use this document to help us back up the program and edit the program more conveniently. In addition, using GitHub also allows us to easily collaborate with others to complete development.
